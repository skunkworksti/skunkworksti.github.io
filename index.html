UC1 · Parsing Operational
Document Parser & Indexer
[Bert] · v1.1

Ingests SOPs, IPIs, and work instructions. Builds a machine-readable knowledge library with provenance tracking and a SPOF register.

  • Dual input modes — institutional docs and operator-fed extracts
  • Hard output gate — never infers content not present in source material
  • Compression hook at 60% context — auto-dumps entries, continues without pause
UC1 · Interview Operational
Knowledge Preservation Interviewer
[Faye] · v1.2

Trust-centered, structured interviews with subject matter experts to capture institutional knowledge before retirement or reassignment.

  • Six risk categories: SPOF, Knowledge Gap, Tribal Knowledge, Undocumented Process, Procedural Delta, Retirement Risk
  • Paul integration — mid-session library cross-reference without interrupting the interview
  • Outbrief is facilitator-only — interviewee receives professional close only
UC1 · Audit Operational
Knowledge Base Auditor
[Joan] · v1.2

Reviews and prunes the cumulative library — detecting drift, flagging gaps, blocking distribution until HIGH-severity issues resolve.

  • Drift detection — flags PROCEDURAL DELTA between institutional and interview-derived entries
  • Blocks FULLREPORT on unresolved HIGH flags (SPOF, pending corroboration)
  • CORROBORATE command verifies entries against source documents
UC2 · QA Operational
QA Incident Reconstruction Agent
[Theseus] · v1.0

Reconstructs QA incidents through structured Team Learning Sessions — deploys immediately after an incident, before the investigation team assembles.

  • 6W framework: What, Who, When, Where, Why, How
  • 4-layer HTML Investigator's Package: Executive Dashboard, Interactive Timeline, Divergence Report, Full Transcripts
  • Captures first accounts while memory is fresh — informs, does not replace, formal QA
Synthesis Operational
Aggregation & Synthesis Agent
[Clew]

Synthesizes outputs across the full agent pipeline into consolidated intelligence products — from interview capture through library audit.

  • Cross-session synthesis of Bert, Faye, and Joan outputs
  • Identifies patterns and gaps spanning the full knowledge library
  • Named for the etymological root of "clue" — finding signal in accumulated record
Workforce · Q&A Operational
Organizational Q&A Agent
[Aletheia]

Human-facing interface for workforce Q&A against the completed knowledge library — making preserved institutional knowledge queryable on demand.

  • Natural language Q&A against the audited knowledge library
  • Retrieves and surfaces relevant entries for workforce queries
  • Named for the Greek word for "truth" — the endpoint of the preservation mission
CUI · Sub-Agent Operational
Derivative Classification Officer
[Lane]

Reviews all agent-generated outputs for CUI compliance before content exits the pipeline for distribution.

  • Sub-agent linked to Bert, Faye, and Theseus via Agent Builder
  • Applies derivative classification rules at the pipeline level
  • Flags CUI handling issues before any output reaches distribution
UC1 · Retrieval Operational
Retrieval Sub-Agent
[Paul] · v1.0

Holds the knowledge library for a work center, surfaces relevant entries into Faye's context at session start and on mid-interview queries.

  • Three-level match logic — exact → partial → SEI
  • Serves Faye mid-interview without interrupting the session flow
  • Zero editorializing — returns entries exactly as filed
Infrastructure Active
The Knowledge Base
[Repository]

The cumulative, structured record of everything the system has captured, indexed, and audited across all work centers and sessions.

  • Schema-locked v1.0 — consistent structure across every entry
  • Grows through Bert parsing and Faye interview output
  • Maintained through Joan's audit cadence — no unreviewed entries reach distribution
